# Настройка промышленного дашборда Grafana

Для обеспечения детального контроля над состоянием четырех ускорителей Tesla V100 используется эталонный шаблон мониторинга. Поскольку стандартные настройки рассчитаны на серверные стойки, требуется ручная адаптация файла под параметры настольного узла NeuralTower.

## 1. Загрузка шаблона
Официальный файл конфигурации необходимо загрузить с портала Grafana Dashboards:
* **Источник:** [NVIDIA DCGM Exporter Dashboard (ID: 12239)](https://grafana.com)
* **Метод:** Нажмите кнопку «Download JSON» и сохраните файл под именем `grafana_dashboard.json` в текущую директорию со скриптами.

## 2. Ручные правки конфигурации
Откройте загруженный файл `grafana_dashboard.json` в любом текстовом редакторе и внесите следующие изменения:

* **Наименование проекта:** 
  Найдите строку `"title": "NVIDIA DCGM Exporter Dashboard"` и замените её на `"title": "NeuralTower Operational Center"`.
* **Частота опроса:** 
  Для мгновенной фиксации скачков температуры при нагрузке в 1.4 кВт найдите параметр `"refresh": "1m"` и измените его на `"refresh": "5s"`.
* **Идентификатор (UID):** 
  Найдите строку `"uid": "..."` и замените её значение на `"uid": "neuraltower_full"`. Это необходимо для корректной работы системы автоматического импорта (Provisioning).
* **Коррекция порогов (Thresholds):**
  Найдите секции, отвечающие за температуру (GPU Temp), и установите критический порог (red) на значение `75`. Для памяти HBM2 на модулях V100 это является предельно допустимым значением при длительной работе.

## 3. Применение изменений
После сохранения правок убедитесь, что в файле `docker-compose.yaml` прописан путь к этому JSON-файлу в секции `volumes` для контейнера Grafana. При следующем запуске команды `docker-compose up -d` дашборд будет импортирован автоматически со всеми внесенными изменениями.
