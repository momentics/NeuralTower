# Настройка программной среды и оптимизация системы

NeuralTower требует специфической конфигурации на уровне BIOS и ядра Linux для обеспечения стабильной работы шины PCIe и управления 128 ГБ видеопамяти HBM2. Данный документ описывает этапы подготовки системы к запуску задач глубокого обучения и инференса.

## 1. Конфигурация BIOS ASUS X99-E WS

Для инициализации четырех ускорителей Tesla V100 и корректного распределения ресурсов необходимо выставить следующие параметры:

*   **Above 4G Decoding:** Enabled. Критически важная настройка для адресации больших объемов видеопамяти.
*   **PCIe Speed:** Force Gen3. Установка фиксированного поколения шины предотвращает ошибки согласования на длинных линках SlimSAS.
*   **PCIe Bifurcation:** x8/x8 для всех слотов, где используются разветвители.
*   **VT-d / Virtualization:** Disabled (если не планируется использование проброса GPU в виртуальные машины) для минимизации задержек IOMMU.

## 2. Параметры загрузки ядра Linux

В файле `/etc/default/grub` необходимо модифицировать строку параметров загрузки для корректного распределения адресов PCIe мостами PLX:

`GRUB_CMDLINE_LINUX_DEFAULT="quiet splash pci=realloc,assign-busses pci=hp_pcie_bus_max"`

После внесения изменений необходимо обновить конфигурацию: `sudo update-grub`.

## 3. Развертывание среды для дообучения моделей

Для трансформации узла NeuralTower из станции инференса в полноценную фабрику по дообучению (Fine-tuning), используется стек на базе Unsloth и Axolotl. Это позволяет эффективно распределять градиенты между четырьмя ускорителями и минимизировать накладные расходы на хранение промежуточных состояний оптимизатора.

### Оптимизация через Unsloth
Интеграция Unsloth позволяет сократить потребление видеопамяти на 40–50% при сохранении точности весов. В условиях 2026 года для архитектуры Broadwell и CUDA 13.x применяется ручная сборка ядер Triton, что обеспечивает прирост скорости вычислений на 25% по сравнению со стандартными методами. При обучении моделей класса Llama 3 (70B) рекомендуется использовать 4-битную квантизацию весов, что оставляет достаточный объем HBM2 для расширения окна контекста до 32 тысяч токенов.

### Оркестрация через Axolotl
Для управления распределенным обучением применяется конфигуратор Axolotl с использованием стратегии FSDP (Fully Sharded Data Parallel). Это позволяет виртуально объединить память всех четырех карт в единый пул. При настройке YAML-конфигурации Axolotl необходимо активировать параметр `sample_packing`, чтобы максимально плотно загружать тензорные ядра V100 и избегать простоев вычислительного конвейера.

## 4. Движок инференса vLLM и механизм Swap-on-NVMe

Для работы с моделями, требующими контекста свыше физического объема памяти GPU, используется движок vLLM с активацией вытеснения кэша на NVMe-накопитель.

### Настройка Swap-пространства
Рекомендуется использовать выделенный NVMe-диск, подключенный напрямую к линиям процессора. Создание быстрого свопа для нужд CUDA:
1. Создание раздела: `sudo mkswap /dev/nvmeXnX`
2. Установка приоритета: в `/etc/fstab` выставить `pri=100` для данного раздела.

### Параметры запуска vLLM
Для распределения модели на все четыре GPU и активации расширенного кэша:
`python -m vllm.entrypoints.openai.api_server --model <name> --tensor-parallel-size 4 --gpu-memory-utilization 0.95 --swap-space 64`
Увеличение `swap-space` до 64 ГБ позволяет обрабатывать сверхдлинные последовательности без риска аварийного завершения процесса из-за нехватки VRAM.

## 5. Мониторинг и тепловой контроль

Поскольку система использует авторскую схему охлаждения V-Core, мониторинг температур VRM и GPU является обязательным условием стабильности. При дообучении моделей нагрузка на цепи питания возрастает на 30% относительно режима инференса.

Критический порог температуры для памяти HBM2 в данном контуре составляет 75°C. В состав NeuralTower входит аналитический стек Prometheus и Grafana, настроенный на частоту опроса 5 секунд. При превышении заданных лимитов автоматика переводит систему нагнетания воздуха в режим максимальной мощности для защиты структурных элементов мезонинов.
