# Настройка программной среды и оптимизация системы

NeuralTower требует специфической конфигурации на уровне BIOS и ядра Linux для обеспечения стабильной работы шины PCIe и управления 128 ГБ видеопамяти HBM2. Данный документ описывает этапы подготовки системы к запуску задач глубокого обучения.

## 1. Конфигурация BIOS ASUS X99-E WS

Для инициализации четырех ускорителей Tesla V100 и корректного распределения ресурсов необходимо выставить следующие параметры:

*   **Above 4G Decoding:** Enabled. Критически важная настройка для адресации больших объемов видеопамяти.
*   **PCIe Speed:** Force Gen3. Установка фиксированного поколения шины предотвращает ошибки согласования на длинных линках SlimSAS.
*   **PCIe Bifurcation:** x8/x8 для всех слотов, где используются разветвители.
*   **VT-d / Virtualization:** Disabled (если не планируется использование проброса GPU в виртуальные машины) для минимизации задержек IOMMU.

## 2. Параметры загрузки ядра Linux

В файле `/etc/default/grub` необходимо модифицировать строку параметров загрузки для корректного распределения адресов PCIe мостами PLX:

`GRUB_CMDLINE_LINUX_DEFAULT="quiet splash pci=realloc,assign-busses pci=hp_pcie_bus_max"`

После внесения изменений необходимо обновить конфигурацию: `sudo update-grub`.

## 3. Оптимизация vLLM и механизм Swap-on-NVMe

Для работы с моделями, требующими контекста свыше 128 ГБ, используется движок vLLM с активацией вытеснения кэша на NVMe-накопитель.

### Настройка Swap-пространства:
Рекомендуется использовать выделенный NVMe-диск. Создание быстрого свопа для нужд CUDA:
1. Создание раздела: `sudo mkswap /dev/nvmeXnX`
2. Установка приоритета: в `/etc/fstab` выставить `pri=100` для данного раздела.

### Параметры запуска vLLM:
Для распределения модели на все 4 GPU и активации расширенного кэша:
`python -m vllm.entrypoints.openai.api_server --model <name> --tensor-parallel-size 4 --gpu-memory-utilization 0.95 --swap-space 32`

## 4. Мониторинг и тепловой контроль

Поскольку система использует авторскую схему охлаждения V-Core, мониторинг температур VRM и GPU является обязательным. В папку `Software/Linux/scripts` добавлен Bash-скрипт, агрегирующий данные через `nvidia-smi`.

Критический порог температуры для GPU V100 в данном контуре - **65°C**. При превышении этого значения автоматика должна переводить вентиляторы в режим 100% мощности (профиль "Emergency").
