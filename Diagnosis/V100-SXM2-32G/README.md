# Диагностика и тестирование карт NVIDIA Tesla V100 SXM2 32GB (4x GPU)

Данный модуль предназначен для автоматизированного аппаратного тестирования, стресс-нагрузки и валидации четырех вычислительных ускорителей **NVIDIA Tesla V100 SXM2 32GB** в составе платформы NeuralTower.

Тестирование инкапсулировано в Docker-контейнер и базируется на утилитах **NVIDIA DCGM (Data Center GPU Manager)** уровня Diagnostic Level 3, а также низкоуровневом анализе состояния памяти HBM2 через `nvidia-smi`.

## Что проверяет этот тест?
1. **Количество GPU:** Строгая проверка наличия ровно 4 видеокарт в системе.
2. **Деградация памяти (Page Retirement):** Анализ заблокированных ячеек памяти HBM2 из-за аппаратных ошибок (EDC/ECC).
3. **DCGM Diagnostic (Level 3):** 
   - Проверка целостности PCIe-интерфейса и шины NVLink.
   - Тестирование подсистемы питания и температурного режима.
   - Стресс-тест ядер CUDA, тензорных ядер (Tensor Cores) и памяти (глубокий таргетированный тест).

## Требования к хост-системе
- Установленный проприетарный драйвер NVIDIA (рекомендуемая версия `535` или выше).
- Установленный и настроенный `nvidia-container-toolkit` для проброса GPU в Docker.
- Поддержка BIOS: Включенные опции **Above 4G Decoding** и **Large BAR Support** (иначе 4 карты по 32GB не инициализируются).

## Инструкция по запуску

### 1. Сборка Docker-образа
Перейдите в директорию с файлами и выполните сборку:
```bash
docker build -t neuraltower-v100-diag .
```

### 2. Запуск тестирования
Для корректного доступа к аппаратным датчикам и всем картам контейнер необходимо запускать с флагами `--gpus all` и `--privileged`:
```bash
docker run --rm --gpus all --privileged neuraltower-v100-diag
```

## Интерпретация результатов
- **Код выхода 0 (Успешно):** Все 4 карты полностью исправны, деградации памяти не обнаружено, тесты DCGM пройдены.
- **Код выхода 1 и более (Ошибка):** Обнаружена неисправность. Внимательно изучите лог контейнера:
  - Если найдены **Retired Pages**, карта имеет скрытый дефект памяти HBM2 и может приводить к ошибкам в вычислениях ИИ.
  - Если DCGM зафиксировал **Fail** на этапах `Integration/Software`, проверьте перегрев карт или проблемы с питанием по линиям 12V/SXM.
