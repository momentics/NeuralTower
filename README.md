# NeuralTower: Автономный вычислительный узел для глубокого обучения и инференса LLM

<img src="./Docs/Images/logo.png" width="100" align="left" alt="Логотип проекта NeuralTower"> NeuralTower представляет собой попытку радикального переосмысления архитектуры мощной рабочей станции, предназначенной для работы с современными языковыми моделями. В условиях, когда объем видеопамяти видеокарт становится критическим ограничением, данная разработка предлагает альтернативный путь через использование потенциала четырех серверных ускорителей NVIDIA Tesla V100 SXM2. Объединение 128 ГБ памяти HBM2 в настольном форм-факторе позволяет развертывать и дообучать модели уровня Llama3 70B полностью локально.

## Инженерная концепция V-CORE и управление тепловыми потоками

Ключевым вызовом при проектировании стала утилизация 1.4 кВт тепловой мощности в условиях жилого помещения. Традиционные методы охлаждения серверных компонентов требуют экстремально высокого давления воздуха, создаваемого шумными турбинами, что неприемлемо для домашней эксплуатации. В NeuralTower эта проблема решена через разделение корпуса на две изолированные зоны горизонтальной алюминиевой палубой. Нижний ярус функционирует как герметичный ресивер давления, где вентиляторы нагнетают воздух сквозь массивные медные радиаторы, формируя так называемый «подвал».

Благодаря герметичности этого объема, воздух принудительно вытесняется через прецизионные сопла в палубе, создавая эффект воздушного ножа. Направленные потоки со скоростью свыше 15 м/с обеспечивают интенсивный обдув цепей питания мезонинов и материнской платы. В то время как графические чипы и центральный процессор находятся под управлением двухконтурной системы жидкостного охлаждения, воздушная составляющая архитектуры V-CORE гарантирует стабильность вторичных компонентов, которые обычно остаются в застойных зонах при использовании водоблоков.

## NVLink и оптимизации под CPU в Gentoo Linux

Для обеспечения беспрецедентной скорости обмена данными внутри вычислительного ядра используется гибридная топология интерконнекта. Пары ускорителей внутри мезонинов объединены мостами NVLink 2.0 с пропускной способностью до 300 ГБ/с, что позволяет рассматривать их как единый вычислительный домен. Связность между мезонинами поддерживается через оптимизированную шину PCIe с активацией технологии GPUDirect P2P, минимизирующей участие центрального процессора в перемещении тензоров между картами.

Программная среда NeuralTower построена на базе Gentoo Linux. Использование этого дистрибутива позволило выполнить полную пересборку системы с оптимизацией под микроархитектуру Broadwell-EP и актуальный стек CUDA 13.x. Это критически важно для работы движка vLLM, который в данной системе дополнен механизмом динамического вытеснения кэша на NVMe-накопитель. Такая программно-аппаратная связка позволяет эффективно работать с контекстами, объем которых кратно превышает физическую емкость видеопамяти.

## Навигация по технической документации

Репозиторий содержит исчерпывающий набор данных для воспроизведения системы, распределенный по этапам реализации. В разделе [Docs/Calculations](./Docs/Calculations) собрана теоретическая база с детальным обоснованием гидродинамических сопротивлений и аэродинамических показателей. Полный реестр вычислительных компонентов, материалов рамы из профиля 2020, фитингов СЖО и метизов зафиксирован в [BOM-листе](./Docs/BOM/bom_list.md). 

Инженерные изыскания по части электропитания, заземления каркаса и изготовления кастомных переходников вынесены в папку [Electrical](./Electrical). Механическая часть, включая руководства по нарезке резьбы в профиле, юстировке рамы и монтажу герметичных панелей, описана в разделах [CAD/Frame](./CAD/Frame), [CAD/Mounts](./CAD/Mounts) и [CAD/Covers](./CAD/Covers). Инструкции по развертыванию MLOps-стека, настройке мониторинга в Grafana и оптимизации операционной среды находятся в папке [Software](./Software).

Подробная история проектирования, описание примененных инженерных подходов и результаты тестов производительности опубликованы в цикле статей в разделе [Articles](./Articles).
