# NeuralTower

<img src="./Docs/Images/logo.png" width="100" align="left" alt="Логотип проекта NeuralTower"> NeuralTower - настольная рабочая станция на четырёх Tesla V100 SXM2 с общим объёмом видеопамяти 128 ГБ. Проект собирается для локального инференса и дообучения моделей уровня Llama3 70B без зависимости от облачной инфраструктуры.
<br clear="left" />

## Зачем

Главное ограничение потребительских видеокарт - объём видеопамяти. Облако решает эту проблему, но быстро съедает бюджет и не всегда подходит для задач, где важны приватность данных и полный контроль над средой.

Четыре V100 SXM2 вместе дают 128 ГБ HBM2. Этого достаточно, чтобы держать в памяти модели, которые на одной RTX 4090 просто не запускаются. Цена такого подхода тоже понятна: это серверное железо, рассчитанное на продув мощными турбинами в стойке. Задача NeuralTower - уместить такую конфигурацию на рабочем столе и не получить вместо вычислительного узла маленький реактивный двигатель.

## Почему V100, если это старое железо

V100 вышла в 2017 году на архитектуре Volta (`sm_70`). По формальным признакам это уже не флагман, но на вторичном рынке у этих карт всё ещё сильное соотношение цены к объёму видеопамяти. 32 ГБ HBM2 на карту с пропускной способностью 900 ГБ/с - это по объёму столько же, сколько у потребительской RTX 5090, хотя по пропускной способности памяти V100 ей уступает: у RTX 5090 она составляет около 1792 ГБ/с.

Недостатки есть, и проект прямо с ними работает:

- **Нет FP8 и MXFP4.** Volta не поддерживает новые форматы квантизации аппаратно. В vLLM фиксируем `--dtype float16`. BF16 тоже работает, но медленнее. Для моделей 70B используем 4-битную квантизацию через AWQ с 1Cat-vLLM или bitsandbytes. Да, это медленнее FP8 на H100, зато модель помещается в память.
- **Официальный vLLM 0.20+ снял поддержку `sm_70`.** Форк 1Cat-vLLM возвращает FlashAttention-2 для Volta и добавляет AWQ 4-bit. Без него vLLM падает на Triton-ядрах (`VLLM_ATTENTION_BACKEND=TRITON_ATTN`). На V100 эта схема работает стабильно, но первый запуск модели дольше из-за JIT-компиляции, поэтому таймаут выставлен в 600 секунд.
- **CUDA 13.x относит Volta к legacy.** Драйверы 580+ всё ещё поддерживают `sm_70`, но без новых оптимизаций. Для 1Cat-vLLM используется CUDA 12.8, которая стабильно работает с V100. Дополнительно Gentoo собирает систему из исходников с флагами `-march=broadwell` и `-flto=auto`, чтобы выжать максимум из доступной платформы.
- **Застревание при инициализации.** В версиях vLLM 0.16.x есть баг с deadlock при запуске некоторых моделей на V100. Обходится принудительным указанием `--max-model-len` и увеличением `VLLM_TENSORRT_LLM_TIMEOUT`.

По сути, V100 - бюджетный способ получить большой объём быстрой памяти. За это приходится платить скоростью генерации токенов, но взамен появляется возможность запустить модель, которая на новых картах с меньшим объёмом VRAM просто не поместится.

## V-CORE

V-CORE - схема охлаждения, где вода и воздух работают вместе, а не мешают друг другу. GPU и CPU охлаждаются жидкостным контуром: пять водоблоков, два медных радиатора 360×60 мм и две помпы D5. Контур сделан последовательно-параллельным: CPU стоит в серии, после чего поток делится на два плеча по мезонинам.

Воздух отвечает за то, что не закрывает вода: VRM материнской платы, цепи питания мезонинов и открытые зоны текстолита. Корпус разделён алюминиевой палубой на два отсека. В нижнем ярусе шесть вентиляторов Noctua NF-A12x25 прогоняют воздух через радиаторы СЖО и создают избыточное давление. Палуба герметична, поэтому воздуху остаётся один путь - через прорези-сопла.

На выходе получается направленный поток со скоростью 15+ м/с. Он бьёт не «примерно внутрь корпуса», а в конкретные горячие зоны. Внизу система остаётся тихой, сверху компоненты получают нужный обдув.

## Связь между GPU

Внутри мезонина карты общаются по NVLink 2.0: до 300 ГБ/с суммарно, задержка около 1,5 мкс. Между мезонинами данные идут по восьми шлейфам SlimSAS через мосты PLX на плате. Здесь уже работает PCIe 3.0 x16, а задержка вырастает примерно до 5 мкс.

GPUDirect P2P включён, поэтому обмен идёт напрямую между GPU и не уходит лишний раз через системную память. На практике такая топология даёт около 70% производительности монолитного серверного решения. Для настольной системы этого достаточно, чтобы комфортно работать с моделями класса 70B.

## Софт

Операционная система - Gentoo Linux, собранная из исходников под Broadwell-EP. Используются CUDA 12.8 и драйверы 580+.

Движок инференса - форк 1Cat-vLLM, потому что официальный vLLM 0.20+ снял поддержку `sm_70`. В форке есть FlashAttention-2 для Volta и AWQ 4-bit. Дополнительно используется вытеснение KV-кэша на NVMe, поэтому рабочий контекст может выходить за пределы физических 128 ГБ видеопамяти.

Для дообучения применяется стек Unsloth + Axolotl с FSDP и 4-битной квантизацией.

Мониторинг построен на Prometheus + Grafana. Опрос выполняется каждые 5 секунд. Критический порог HBM2 выставлен на 75 °C.

## Быстрый старт мониторинга

```bash
cd Software/Linux/scripts
docker-compose up -d
./monitor.sh
```

Скрипты позволяют отслеживать температуры HBM2, GPU, VRM и потребление каждого модуля в реальном времени. Для постоянного наблюдения используется Grafana.
