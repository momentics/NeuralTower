# NeuralTower: Автономный вычислительный узел для глубокого обучения и инференса LLM

Проект NeuralTower представляет собой попытку радикального переосмысления архитектуры мощной рабочей станции, предназначенной для работы с современными языковыми моделями. В условиях, когда объем видеопамяти потребительских видеокарт становится критическим ограничением, данная разработка предлагает альтернативный путь через использование потенциала четырех серверных ускорителей NVIDIA Tesla V100 SXM2. Объединение 128 ГБ сверхбыстрой памяти HBM2 в настольном форм-факторе позволяет развертывать и дообучать модели уровня Llama 3 70B полностью локально, сохраняя конфиденциальность данных и независимость от облачных провайдеров.

## Инженерная концепция V-CORE и управление тепловыми потоками

Ключевым вызовом при проектировании стала утилизация 1.4 кВт тепловой мощности в условиях жилого помещения. Традиционные методы охлаждения серверных компонентов требуют экстремально высокого давления воздуха, создаваемого шумными турбинами, что неприемлемо для домашней эксплуатации. В NeuralTower эта проблема решена через разделение корпуса на две изолированные зоны горизонтальной алюминиевой палубой. Нижний ярус функционирует как герметичный ресивер давления, где вентиляторы нагнетают воздух сквозь массивные медные радиаторы, формируя так называемый «подвал».

Благодаря герметичности этого объема, воздух принудительно вытесняется через прецизионные сопла в палубе, создавая эффект воздушного ножа. Направленные потоки со скоростью свыше 15 м/с обеспечивают интенсивный обдув цепей питания мезонинов и материнской платы. В то время как графические чипы и центральный процессор находятся под управлением двухконтурной системы жидкостного охлаждения, воздушная составляющая архитектуры V-CORE гарантирует стабильность вторичных компонентов, которые обычно остаются в застойных зонах при использовании водоблоков.

## Синергия NVLink и программной оптимизации Gentoo Linux

Для обеспечения беспрецедентной скорости обмена данными внутри вычислительного ядра используется гибридная топология интерконнекта. Пары ускорителей внутри мезонинов объединены мостами NVLink 2.0 с пропускной способностью до 300 ГБ/с, что позволяет рассматривать их как единый вычислительный домен. Связность между мезонинами поддерживается через оптимизированную шину PCIe с активацией технологии GPUDirect P2P, минимизирующей участие центрального процессора в перемещении тензоров между картами.

Программная среда NeuralTower построена на базе Gentoo Linux образца 2026 года. Использование этого дистрибутива позволило выполнить полную пересборку системы с оптимизацией под микроархитектуру Broadwell-EP и актуальный стек CUDA 13.x. Это критически важно для работы движка vLLM, который в данной системе дополнен механизмом динамического вытеснения кэша на NVMe-накопитель. Такая программно-аппаратная связка позволяет эффективно работать с контекстами, объем которых кратно превышает физическую емкость видеопамяти.

## Навигация по технической документации

Репозиторий содержит исчерпывающий набор данных для воспроизведения системы, распределенный по этапам реализации. В разделе **[Docs/Calculations](./Docs/Calculations)** собрана теоретическая база с детальным обоснованием гидродинамических сопротивлений и аэродинамических показателей. Инженерные изыскания по части электропитания, заземления каркаса и изготовления кастомных переходников вынесены в папку **[Electrical](./Electrical)**. Механическая часть, включая руководства по нарезке резьбы в профиле, юстировке рамы и монтажу герметичных панелей, описана в разделах **[CAD/Frame](./CAD/Frame)**, **[CAD/Mounts](./CAD/Mounts)** и **[CAD/Covers](./CAD/Covers)**. Инструкции по развертыванию MLOps-стека, настройке мониторинга в Grafana и оптимизации операционной среды находятся в папке **[Software](./Software)**.

## Эксплуатация и мониторинг

Система поставляется с готовым набором инструментов для контроля параметров в реальном времени. Развернутый аналитический комплекс позволяет отслеживать температуру памяти HBM2 и энергопотребление каждого GPU, обеспечивая безопасность работы узла при длительных сессиях обучения или инференса.

```bash
cd Software/Linux/scripts
docker-compose up -d  # Развертывание полного стека сервисов
./monitor.sh          # Запуск локальной консоли мониторинга
```

Подробная история проектирования, описание примененных инженерных подходов и результаты тестов производительности опубликованы в цикле статей в разделе **[Articles](./Articles)**.
