# NeuralTower

<img src="./Docs/Images/logo.png" width="100" align="left" alt="Логотип проекта NeuralTower"> Настольная рабочая станция на четырёх Tesla V100 SXM2 с общим объёмом видеопамяти 128 ГБ. Собирается для локального инференса и дообучения моделей вроде Llama3 70B без облаков.
<br clear="left" />

## Зачем

Потребительские карты упираются в объём видеопамяти. Облако жрёт бюджет и не даёт приватности. Четыре V100 SXM2 вместе дают 128 ГБ HBM2 - этого хватает, чтобы держать в памяти модели, которые на RTX 4090 даже не запускаются. Минус в том, что это серверное железо, рассчитанное на продув турбинами в стойке. Задача - уместить всё на столе и не получить реактивный двигатель.

## Почему V100, если это старое железо

V100 вышел в 2017 году на архитектуре Volta (sm_70). По бумагам они уже не тянут на флагманов, но на вторичном рынке соотношение цена/ГБ памяти остаётся одним из лучших. 32 ГБ HBM2 на карту с пропускной способностью 900 ГБ/с — это столько же по объёму, сколько у потребительской RTX 5090, хотя и уступает ей по пропускной способности памяти (у RTX 5090 она составляет ~1792 ГБ/с).

Есть недостатки, и проект с ними работает:

- **Нет FP8 и MXFP4.** Volta не поддерживает новые форматы квантизации аппаратно. В vLLM фиксируем `--dtype float16`, BF16 тоже работает, но медленнее. Для моделей 70B используем 4-битную квантизацию через AWQ (с 1Cat-vLLM) или bitsandbytes — да, это медленнее FP8 на H100, но модель влезает в память.
- **Официальный vLLM 0.20+ снял поддержку sm_70.** Форк 1Cat-vLLM восстанавливает FlashAttention-2 для Volta и добавляет AWQ 4-bit. Без него vLLM падает на Triton-ядра (`VLLM_ATTENTION_BACKEND=TRITON_ATTN`), что на V100 работает стабильно, просто компиляция JIT при первом запуске модели занимает дольше — ставим таймаут 600 секунд.
- **CUDA 13.x помечает Volta как legacy.** Драйверы 580+ всё ещё поддерживают sm_70, но без новых оптимизаций. Для 1Cat-vLLM используем CUDA 12.8, которая стабильно работает с V100. Обходной путь: Gentoo собирает всё из исходников с флагами `-march=broadwell` и `-flto=auto`, выжимая максимум из того, что есть.
- **Застревание при инициализации.** В версиях vLLM 0.16.x есть баг с deadlock при старте некоторых моделей на V100. Фиксится принудительным указанием `--max-model-len` и увеличением `VLLM_TENSORRT_LLM_TIMEOUT`.

По сути, V100 — это бюджетный способ получить большой объём быстрой памяти. Мы платим за это скоростью токенов в секунду, но получаем возможность запустить модель, которая на новых картах просто не влезет в VRAM одной штуки.

## V-CORE

V-CORE - это схема охлаждения, которая объединяет воду и воздух в одном корпусе. Сами GPU и CPU охлаждаются водой: пять водоблоков, два медных радиатора 360×60 мм, две помпы D5. Контур последовательно-параллельный - CPU в серии, потом поток делится на два плеча по мезонинам.

Воздух берёт на себя то, что вода не покрывает: VRM материнской платы, цепи питания мезонинов, открытые зоны текстолита. Корпус разрезан алюминиевой палубой на два отсека. В нижнем ярусе шесть вентиляторов Noctua NF-A12x25 нагнетают воздух через радиаторы СЖО, создавая избыточное давление. Палуба герметична, и воздуху некуда деться, кроме как через прорези-сопла. На выходе получается поток 15+ м/с, направленный точно в горячие зоны. Внизу тихо, сверху прохладно.

## Связь между GPU

Внутри мезонина карты общаются по NVLink 2.0 - 300 ГБ/с суммарно, задержка около 1.5 мкс. Между мезонинами данные идут по восьми шлейфам SlimSAS через мосты PLX на плате, PCIe 3.0 x16, задержка вырастает до 5 мкс. GPUDirect P2P включён, так что обмен идёт напрямую между GPU, не проходя через системную память. На практике это даёт примерно 70% от производительности монолитного серверного решения, чего хватает для комфортной работы с моделями 70B.

## Софт

Gentoo Linux, всё собрано из исходников под Broadwell-EP. CUDA 12.8, драйверы 580+. Движок инференса — форк 1Cat-vLLM (официальный vLLM 0.20+ снял поддержку sm_70) с FlashAttention-2 для Volta и AWQ 4-bit, плюс вытеснение KV-кэша на NVMe, так что рабочий контекст может превышать физические 128 ГБ. Для дообучения используется стек Unsloth + Axolotl с FSDP и 4-битной квантизацией.

Мониторинг на Prometheus + Grafana, опрос каждые 5 секунд. Критический порог HBM2 выставлен на 75 °C.

## Быстрый старт мониторинга

```bash
cd Software/Linux/scripts
docker-compose up -d
./monitor.sh
```

Отслеживает температуры HBM2, GPU, VRM и потребление каждого модуля в реальном времени (это консоль. не забываем про Grafana).
